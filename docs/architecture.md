## 2. 技術スタック

| レイヤー | 技術 | 理由 |
|---------|------|------|
| **フロントエンド** | Next.js (静的エクスポート) | Cloudflare Pagesの無料枠で配信可能 |
| **UIライブラリ** | shadcn/ui + Tailwind CSS | 高品質UIを効率的に構築 |
| **バックエンド** | FastAPI (Python) | AI/ML系ライブラリとの相性、async対応 |
| **PDF解析** | PyMuPDF (fitz) | テキスト抽出+ページ画像化の両方が可能 |
| **AI台本生成** | Google Gemini / OpenAI GPT | ユーザーのAPIキーで呼び出し |
| **音声合成（無料）** | Edge-TTS | Microsoft Edge読み上げエンジン。無料・高品質・日本語対応 |
| **音声合成（有料）** | OpenAI TTS | ユーザーのAPIキーで呼び出し。より自然な音声 |
| **動画生成** | FFmpeg (subprocess) | 画像+音声→MP4。moviepyより直接制御で安定 |
| **リアルタイム通信** | Server-Sent Events (SSE) | 進捗表示。WebSocketより実装がシンプル |
| **Webサーバー** | Nginx | リバースプロキシ、SSL終端、静的ファイル配信 |
| **SSL** | Let's Encrypt (certbot) | 無料SSL証明書 |
| **VPS** | ConoHa VPS / さくらVPS 等 | 月500〜1,000円のプラン |
| **ドメイン** | Cloudflare Registrar 等 | 年1,000〜2,000円 |

## 4. Edge-TTS について

Edge-TTSはMicrosoft Edgeの読み上げエンジンをPythonから利用するライブラリ。

### メリット
- 完全無料（APIキー不要）
- 高品質な日本語音声（複数話者対応）
- 速度・ピッチの調整が可能
- async対応で非同期処理と相性が良い

### デメリット
- Microsoftのサービスに依存（将来的に仕様変更の可能性）
- OpenAI TTSほどの表現力はない

### 利用可能な日本語音声（一部）

## 5. バックエンドの処理フロー詳細

    POST /generate (PDF + APIキー)
        │
        ▼
    [1. ジョブ作成]
        │  job_id を発行、ステータスを "processing" に
        │  バックグラウンドタスクとして処理開始
        │  即座に { job_id } をレスポンス
        │
        ▼
    [2. PDF解析] ── 進捗 10%
        │  PyMuPDFでページごとに:
        │  - テキスト抽出
        │  - ページ画像をPNGとして保存
        │
        ▼
    [3. AI台本生成] ── 進捗 10〜50%
        │  スライドごとに:
        │  - テキスト+画像をAI APIに送信
        │  - 台本テキストを取得
        │  （ユーザーのAPIキーを使用）
        │
        ▼
    [4. 音声合成] ── 進捗 50〜80%
        │  スライドごとに:
        │  - 台本テキストをEdge-TTSに送信
        │  - MP3音声ファイルを保存
        │
        ▼
    [5. 動画合成] ── 進捗 80〜95%
        │  FFmpegで:
        │  - 各スライド画像を音声の長さ分表示
        │  - 全スライドを結合
        │  - MP4として出力
        │
        ▼
    [6. 完了] ── 進捗 100%
        │  ダウンロードURLをSSEで通知
        │
        ▼
    [7. クリーンアップ] ── 30分後
           一時ファイル（PDF, 画像, 音声, 動画）を削除
